=================================================================
PropEdge April 4 Sprint - Quick Testing Guide (5 min)
=================================================================

BEFORE DEPLOYMENT:
1. Open browser DevTools (Cmd+Option+I on Mac)
2. Go to Console tab
3. Copy-paste commands below to test each feature

=================================================================
TEST 1: OneSignal Alerts System
=================================================================

Run this in Console:
  window.sentAlerts
  
Expected output:
  Set(0) { }  (empty initially)

Wait 60 seconds, then run again:
  window.sentAlerts

Expected: 
  May have 1+ entries if pace-UP players detected

Look for console messages like:
  [PropEdge] OneSignal alerts initialized
  [PropEdge Alert] Sent: [player name]

=================================================================
TEST 2: Sportsbook ROI Tracker
=================================================================

View current ROI data:
  JSON.parse(localStorage.getItem('propEdgeROI') || '[]')

Expected:
  Empty array [] initially

Manually simulate a click (test only):
  trackBetClick({player:'Test Player',league:'NBA',line:10,direction:'OVER',edge:5})
  
Then check ROI again:
  JSON.parse(localStorage.getItem('propEdgeROI'))

Expected:
  Array with 1 entry showing test click data

=================================================================
TEST 3: Injury Status Tracking
=================================================================

Check stored status snapshots:
  JSON.parse(localStorage.getItem('propEdgeLastStatus'))

Expected output:
  Object like:
  {
    "1": "Active",
    "2": "Active",
    "27": "Limited",  ← Jaren Jackson Jr.
    ...
  }

TEST 5: Odds Snapshot System
=================================================================

Check if snapshot logger is ready:

(Snapshots start collecting immediately on page load)
